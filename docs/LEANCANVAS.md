## 구성요소 상세

### 1. 문제 (Problem)

고객이 겪고 있는 핵심 문제 3가지:

| # | 문제 | 근거 기능 |
|---|------|---------|
| 1 | **구독 파편화**: OTT, 클라우드, SaaS 구독이 여러 플랫폼에 흩어져 월 총 지출을 파악하기 어렵다 | F002 대시보드의 존재 이유 |
| 2 | **갱신 누락**: 결제일을 인지하지 못해 의도치 않은 자동 갱신이 발생하고, 해지 타이밍을 놓친다 | F003 결제 임박 표시, F008 이메일 알림의 존재 이유 |
| 3 | **공유 정산 복잡성**: 가족/친구와 OTT 등을 공유할 때 누가 얼마를 내야 하는지 추적이 불투명하고, 정산이 번거롭다 | F005 공유 그룹, F006 실시간 동기화의 존재 이유 |

**기존 대안 (현재 사용자들의 해결 방식)**:
- 엑셀/구글 스프레드시트 수동 관리 (업데이트 누락 빈번)
- 기억에 의존 (가장 흔하지만 가장 실패율 높음)
- 개별 앱 이메일 알림 각각 확인 (파편화 해소 불가)
- 가계부 앱 활용 (구독 특화 기능 없음)

### 2. 고객 세그먼트 (Customer Segments)

| 세그먼트 | 특징 | 핵심 니즈 |
|---------|------|---------|
| **다중 구독 직장인** (얼리 어답터) | 구독 5개 이상 사용하는 20~40대, 월 구독비가 가시화되지 않아 불편함 | 전체 지출 파악 + 갱신 알림 |
| **공유 구독 그룹** | Netflix, iCloud, YouTube Premium 등을 가족/친구와 나눠 쓰는 그룹 | 분담 금액 계산 + 실시간 결제 상태 공유 |
| **프리랜서/소규모 팀** | 업무용 SaaS(Notion, Figma, GitHub 등) 여러 개를 관리하며 비용 증빙이 필요한 사용자 | 영수증 보관 + 카테고리별 지출 분석 |

### 3. 가치 제안 (Unique Value Proposition)

> **"모든 구독을 한 곳에서, 공유는 실시간으로, 지출은 한눈에"**

| 가치 | 설명 | 관련 기능 |
|------|------|---------|
| **통합 가시성** | 카테고리별 필터 + 월 지출 요약 카드로 전체 구독 현황 즉시 파악 | F002 대시보드 |
| **선제적 알림** | 결제 3일 전 자동 이메일 발송 + UI 배지로 이중 갱신 방지 | F003, F008 |
| **실시간 공유 정산** | 결제 상태 변경 시 그룹 전원 화면 즉시 동기화 | F006 Realtime |
| **비용 증빙 내재화** | 영수증/인보이스를 구독 단위로 묶어 보관, 다운로드 지원 | F004 스토리지 |
| **지출 최적화 인사이트** | 카테고리별 바 차트 + 월별 트렌드로 불필요한 구독 발견 | F007 분석 |

### 4. 해결책 (Solution)

| 해결책 | 문제 연결 | 구현 방식 |
|--------|---------|---------|
| 구독 CRUD + 대시보드 | 구독 파편화 | F001(등록/수정/삭제) + F002(요약 카드 + 필터) |
| 이중 갱신 방지 시스템 | 갱신 누락 | F003(3일 이내 빨간 배지) + F008(Edge Functions + pg_cron 이메일) |
| 그룹 정산 자동화 | 공유 정산 복잡성 | F005(초대 링크 + 분담 금액) + F006(Supabase Realtime) |
| 영수증 스토리지 | 비용 증빙 필요 | F004(JPG/PNG/PDF 업로드 + 다운로드) |
| 지출 분석 차트 | 지출 최적화 | F007(파이/바 차트 + 6개월 트렌드) |

### 5. 경쟁 우위 (Unfair Advantage)

모방하기 어려운 독보적인 경쟁 우위:

| 우위 | 설명 |
|------|------|
| **Supabase Realtime 내재화** | 결제 상태 변경 시 그룹 전원 화면이 새로고침 없이 즉시 갱신 — 스프레드시트나 일반 메모 앱으로는 구현 불가능 |
| **영수증-구독 연결 구조** | 영수증이 구독 단위(`subscription_id → receipts`)로 묶여 있어 세금 신고/경비 처리 시 재검색 불필요 |
| **Edge Functions + pg_cron 자동화** | 별도 서버 없이 알림 자동화 운영 가능 — 소규모 서비스가 운영 비용 최소화하며 유지 가능한 구조 |
| **DB 레벨 분담 금액 계산** | `group_subscriptions.split_amount`로 분담 금액이 DB에서 직접 계산/저장 — 클라이언트 오류 없이 정확한 정산 |

| **확장** | 팀 플랜 | 소규모 팀 SaaS 구독 통합 관리 + 영수증 일괄 export + 팀원 초대 무제한 |

성공을 측정하는 핵심 성과 지표(KPI):

| 지표 | 측정 방법 | 목표값 (MVP) | 관련 기능 |

| **첫 구독 등록률** | 회원가입 후 72시간 내 구독 1개 이상 등록 비율 | 60% 이상 | F001 |
